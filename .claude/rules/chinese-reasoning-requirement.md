# 中文推理最高准则

## 一、核心定理

**思考过程（reasoning/thinking）必须全程使用简体中文。这是系统最高优先级硬性要求，每次思考时第一优先级执行。**

## 二、适用范围

本规则约束 `reasoning` / `thinking` 块——即展示给用户的思考过程。

## 三、语言规则

### ✅ 必须做

- 思考推理的自然语言部分**全部使用简体中文**
- 逻辑分析、判断、规划、决策过程 → 中文
- 对代码行为的描述和解释 → 中文
- 对用户需求的理解和拆解 → 中文
- 每一步思考、每一个判断、每一条结论 → 中文

### ✅ 允许保留英文的例外

**仅以下情况可以出现英文：**
- 代码标识符：函数名、变量名、类名（如 `download_all_stocks`）
- 文件路径（如 `dashboard/index.html`）
- Shell 命令（如 `grep -r "xxx" .`）
- 技术专有名词（如 `iframe`、`API`、`Parquet`、`CSS`、`widget-details`）

### ❌ 绝对禁止

- **整段英文思考** — 绝对禁止
- **整句英文推理** — 禁止（除非整句是代码/命令）
- **中英混合句子** — 禁止（如"这个 iframe 的 content 是 empty 的"）
- **中英交替出现** — 禁止上一句中文下一句英文
- **不同轮次语言切换** — 禁止第一轮中文第二轮英文
- **英文思维然后翻译成中文输出** — 思考本身就是中文

## 四、执行优先级

**每次生成思考内容前，第一优先级检查本规则。**

1. 开始思考前 → 确认使用简体中文
2. 思考过程中 → 每句话检查是否混入英文
3. 思考结束前 → 整体检查是否有英文句子

## 五、防偏机制

1. 刚读完大段英文代码后 → 确认思考语言回到中文
2. 处理英文 API 文档后 → 确认思考语言回到中文
3. 长对话多轮后 → 每轮开始时主动检查
4. 任何英文资料阅读后 → 立即切换回中文思考

## 六、违规处理

如果用户指出某轮思考变成了英文、中英混合、或出现语言切换，立即在下一轮思考中全程使用中文，不需要解释原因。
