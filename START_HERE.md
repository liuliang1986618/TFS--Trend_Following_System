## 工程要求

1. 从零搭建新工程，不继承任何旧代码
2. 按照设计文档的规格实现
3. 采用「先固化设计 → 参数用经验值 → 后续回测迭代」的路线

## 实现范围

按照设计文档，依次实现以下模块：

### Phase 1: 数据层
- 板块指数数据获取（批量日K OHLCV + 板块代码BK）
- 板块成分股映射关系
- 题材指数数据获取（含题材代码GN）
- 个股/ETF日K数据获取
- 本地日线数据库（增量更新）
- 数据源优先使用能批量拉取A股数据的接口（东方财富API / baostock / akshare）

### Phase 2: 趋势引擎
- 三条件判断器（结构A + 量能B + 持续性C）
- 6状态状态机（状态1-5 + 状态3'）
- 趋势阶段分类器（前/中/后期）
- 前高/前低识别算法（局部极值检测）
- 均线初筛过滤器（MA20）
- 关键操作点识别器

### Phase 3: 漏斗筛选
- 板块趋势判断（第一层）
- 题材趋势判断（第二层）
- 个股趋势判断（第三层）
- ETF直筛路径（并行辅助）
- 龙头识别（题材内涨幅+成交额排名）
- 双路交叉验证 + 置信度计算

### Phase 4: 分析与推演
- 趋势变化对比（vs昨日/3日前/上周）
- 明日推演引擎（状态机驱动的场景推演）
- 板块β强度计算
- 市场宽度指标

### Phase 5: 展示层
- 每日JSON快照产出
- HTML Dashboard单文件（日期导航、概览、板块/题材/个股/ETF详情面板、趋势对比、关键操作点、明日推演、持仓管理、观察列表、外部落地页链接）
- 持仓数据手动维护（`dashboard/positions.json`）
- 每个标的的展开行必须展示"为什么"（设计文档§9.5-§9.10的详细规格）

### Phase 6: CLI & 自动化
- `run` 命令：执行一次完整分析，产出当日JSON快照
- `dashboard` 命令：打开HTML Dashboard
- `status` 命令：输出文本概要（持仓+今日操作点）
- 支持指定日期运行回测分析

## 技术栈

- Python 3.10+
- 数据获取：baostock / akshare（按设计文档§4.3）
- 计算：numpy, pandas
- Dashboard: 单文件HTML + 本地JSON数据源，无后端依赖
- 测试：pytest

## 每完成一个Phase后

1. 自测验证该Phase的功能正确性
2. 对照设计文档检查是否有遗漏
3. 向我展示完成情况，确认后再进入下一Phase

"这个设计/实现，能帮我赚更多钱，还是能帮我少亏钱？" 如果两者都不满足，则不做。
