# 趋势检测系统设计文档

> ## ══════════════════════════════════════════
> ## 唯一准则
> 
> **我是一个趋势交易者。**
> 
> **我需要跟随趋势操作，保证我能赚钱。**
> 
> **赚钱是一切的初衷和目标，是所有行为准则，**
> **是唯一不能丝毫动摇的标准，是唯一的追求。**
> 
> ---
> 
> 每一个功能、每一个参数、每一行代码、每一个判断逻辑，无论大小，在实现前必须回答：
> 
> **"这个设计/实现，能帮我赚更多钱，还是能帮我少亏钱？"**
> 
> 如果两者都不满足，则不做。

> ## 1. 核心定义: 什么是上涨趋势

上涨趋势 = 价格上涨, 且同时满足结构、量能、持续性三个独立维度的验证。

**任何一个不满足就不叫上涨趋势, 哪怕价格在涨。**

### 1.1 三个必要条件 (缺一不可)

| 条件 | 标准 | 验证的维度 |
|------|------|-----------|
| A. 结构 | 至少1对更高高+更高低(前期) / 2对(中期确认) | 方向: 价格是否真的在建立新高 |
| B. 量能 | 近20日: 上涨日平均成交量 > 下跌日平均成交量 | 力量: 是真金白银推动还是无量空涨 |
| C. 持续性 | 近20日: 阳线>阴线, 且出现过连续3根阳线 | 节奏: 多头是否持续占据主导 |

### 1.2 辅助条件 (不参与趋势确认, 仅用于初筛)

| 条件 | 标准 | 用途 |
|------|------|------|
| D. 均线过滤 | 价格 > MA20 | 快速排除无效标的, 减少计算量 |

### 1.3 设计原则

- 每个条件验证一个独立维度, 互不冗余
- 结构是准入条件, 量能+持续性是确认条件
- 对股票/ETF/板块指数/题材指数统一适用同一套标准

## 2. 状态机: 6种核心状态

### 2.1 状态定义

#### 状态1: 下跌趋势

- 低点持续降低 (至少2个更低低点)
- 高点持续降低 (至少2个更低高点)
- 价格在 MA20 下方运行
- 量能特征: 下跌放量, 反弹缩量
- **仓位: 0, 空仓。唯一动作: 纳入观察列表**

#### 状态2: 下跌中的反弹

- 在状态1中, 价格出现连续上涨
- 但未突破前一个高点 (下跌结构未被打破)
- 量能特征: 缩量上涨为主
- **仓位: 0, 不动。盯紧前高, 看能否突破**

#### 状态3: 翻转确认中 (下跌→上涨)

- 价格放量突破了前一个高点 (打破下跌结构)
- 出现了一个更高的低点 (回调不破前低)
- 但完整上涨结构尚未形成 (还没有第2个更高的高点)
- **仓位: 1/6 ~ 1/3 试探仓, 严格止损**

#### 状态4: 上涨趋势

- 2个更高高 + 2个更高低 (完整多头结构)
- 量能: 上涨日平均量 > 下跌日平均量
- 持续性: 阳线 > 阴线, 有过3连阳
- 价格在 MA20 上方
- **仓位: 标准仓 (100%), 持股为主**

#### 状态5: 上涨中的回调

- 在状态4中, 价格出现连续下跌
- 未跌破前一个低点 (上涨结构未被破坏)
- 量能特征: 缩量下跌 (正常回调)
- **仓位: 标准仓(不动), 等企稳加仓至1.2~1.5倍**

#### 状态3': 转跌确认中 (上涨→下跌, 对称于状态3)

- 价格放量跌破了前一个低点 (打破上涨结构)
- 出现了一个更低的高点 (反弹不过前高)
- 但完整下跌结构尚未形成 (还没有第2个更低的低点)
- **仓位: 1/3, 止损线紧贴前低, 等待确认方向**

### 2.2 状态转换与操作

| 转换 | 触发条件 | 仓位动作 | 性质 |
|------|---------|---------|------|
| 1→2 | 出现连续上涨, 缩量未破前高 | 不动, 盯前高 | 观察 |
| 2→1 | 反弹结束, 回落跌破前低 | 继续空仓 | 延续 |
| **2→3** | **放量突破前高** | **1/6 试探进场** | **买点1** |
| **3→1** | **假突破, 跌破新低** | **全清止损** | **止损** |
| 3→3 | 回调不破前低 | 加至 1/3 | 加仓 |
| **3→4** | **再创新高, 结构完整** | **加至标准仓(100%)** | **买点2** |
| 4→4 | 继续新高 | 持仓不动 | 持股 |
| 4→5 | 出现连续下跌, 缩量未破前低 | 持仓, 盯前低 | 观察 |
| **5→4** | **企稳反弹, 不破前低** | **加至 1.2~1.5 倍** | **最佳加仓** |
| **5→3'** | **放量跌破前低** | **减至 1/3** | **防守** |
| 3'→4 | 假跌破, 快速收复 | 回补标准仓 | 修复 |
| **3'→1** | **再破新低, 结构破坏** | **全部清仓** | **退场** |

加粗行 = 需要用户决策的关键操作点 (共6个)

### 2.3 关键术语定义

- **前高/前低**: 指价格走势中最近一个明显的局部最高点/最低点。一个局部高/低点定义为: 该日最高价/最低价比前后各3日的都高/低。若该点距今超过60个交易日, 则认为已失效, 参考次近的点
- **连续上涨**: 至少2个交易日连续收阳, 且累计涨幅 > 2%。单日孤立反弹不计为连续上涨
- **连续下跌**: 至少2个交易日连续收阴, 且累计跌幅 > 1.5%。单日孤立回调不计为连续下跌
- **放量**: 当日成交量 > 近20日均量的1.2倍
- **缩量**: 当日成交量 < 近20日均量的0.8倍

### 2.4 趋势阶段映射

```
前期 (状态3):   结构初步形成, 试探建仓, 胜率低但赔率高
中期 (状态4早期): 结构完整确认, 标准仓位, 主升浪持股
后期 (状态4晚期): 斜率变陡 + 放量滞涨 + 回调频繁
                  → 收紧止损, 不加仓, 不减仓
                  → 等待状态5→3'→1 的退场信号
```

## 3. 漏斗筛选架构

### 3.1 整体数据流

```
全市场 (5000+ 标的)
    │
    ▼ 第一层: 板块趋势判断 (~60~90个行业板块指数)
    │   ├─ 输入: 板块日K OHLCV
    │   ├─ 逻辑: 跑状态机, 筛出状态3/4/5的板块
    │   └─ 输出: 上涨板块列表 (约5~15个)
    │
    ▼ 第二层: 题材趋势判断 (上涨板块内的题材指数)
    │   ├─ 输入: 题材日K OHLCV
    │   ├─ 逻辑: 跑状态机, 筛出状态3/4的题材
    │   └─ 输出: 活跃题材列表 (约3~8个)
    │
    ▼ 第三层: 个股/ETF趋势判断 (活跃题材内的成分股)
    │   ├─ 输入: 个股日K OHLCV (~300~500只)
    │   ├─ 逻辑: 跑状态机, 输出状态+关键点信号
    │   └─ 输出: 趋势个股 + 交易信号 (约5~20只)
    │
    ▼ 第四层: 交易决策
        ├─ 状态标签 + 阶段标签 + 仓位建议
        └─ 只在6个关键操作点发出信号
```

### 3.2 辅助路径: ETF 直筛

与漏斗主路径并行运行:

```
ETF 池 (~100~200只)
    │
    ▼ 直接跑状态机 (不经过板块→题材漏斗)
    │
    ▼ 输出: 自身处于上涨趋势的 ETF
    │   ├─ 类型A (严格对应板块): 板块ETF, 板块确认后直接可交易
    │   ├─ 类型B (跨板块/题材): 通信ETF、新能源ETF等, 独立判断
    │   └─ 类型C (宽基/策略): 沪深300ETF、红利ETF等, 不适用本策略, 跳过
    │
    ▼ 交叉验证
        ├─ 漏斗命中 + ETF命中 = 最高置信度, 标准仓位
        ├─ 仅漏斗命中 = 标准置信度, 标准仓位
        └─ 仅ETF直筛命中 = 中置信度, 试探仓位
```

### 3.3 最终输出标的分类

| 风险偏好 | 交易标的 | 来源路径 | 弹性 |
|---------|---------|---------|------|
| 稳健 | 行业ETF / 主题ETF | 漏斗+直筛 | 低, 跟板块 |
| 平衡 | 板块内核心权重股 | 漏斗 | 中 |
| 激进 | 题材龙头弹性股 | 漏斗 | 高 |

## 4. 数据获取策略

### 4.1 每日数据需求

| 数据 | 数量 | 接口类型 | 预估耗时 |
|------|------|---------|---------|
| 板块指数日K | 60~90个 | 批量接口 | <1秒 |
| 上涨板块成分股日K | 300~500只 | 批量接口 | <5秒 |
| ETF列表+日K | 100~200只 | 批量接口 | <2秒 |
| 题材指数日K | 按需 | 批量接口 | <1秒 |
| **合计** | **~500~600条** | | **<10秒** |

### 4.2 本地数据维护

- 维护本地日线数据库, 每日增量追加 (仅当日数据), 不重复拉历史数据
- 历史数据首次初始化时一次性全量拉取
- 板块→成分股映射关系定期更新 (每周一次)

### 4.3 数据源选择

A股日频OHLCV即可满足需求, 推荐使用支持批量获取的接口:

- 优先: 东方财富等支持一次性全市场或批量拉取的接口
- 备选: baostock (query_all_stock)、tushare (批量接口)
- 关键要求: 必须支持按板块查成分股、批量拉取日K

## 5. 输出格式

### 5.1 每只标的输出

```
{
  symbol: "000001",
  name: "平安银行",
  type: "stock" | "etf" | "sector_index" | "theme_index",

  state: 1|2|3|4|5,           // 当前状态
  stage: "early"|"mid"|"late", // 趋势阶段 (仅状态3/4时有效)
  stage_label: "前期"|"中期"|"后期",

  conditions: {
    structure:  { pass: true,  detail: "2更高高+2更高低" },
    volume:     { pass: true,  detail: "上涨均量>下跌均量 1.3x" },
    persistence:{ pass: true,  detail: "近20日阳14/阴6, 最大连阳5天" },
  },

  is_key_point: true|false,    // 是否在6个关键操作点
  key_action: "加仓至标准仓",   // 如果是关键点, 给出具体操作

  position: {
    current: 0.33,             // 当前建议仓位比
    target:  1.0,              // 目标仓位比
    stop_loss_price: 12.50,    // 止损价 (基于前低)
  },

  confidence: 0.85,            // 置信度 (综合考虑β支撑+信号强度)
}
```

### 5.2 每日汇总输出

- 上涨趋势中的板块列表 (按强度排序)
- 6个关键操作点上的标的列表 (按紧急程度排序)
- 仓位变动提醒 (哪些标的今天该加仓/减仓/清仓)

## 6. 参数表 (经验值, 待回测校准)

| 参数 | 当前值 | 说明 |
|------|--------|------|
| 结构确认窗口 | 2对高低点 | 状态3→4的确认标准 |
| 量能比较窗口 | 近20日 | 涨跌均量对比周期 |
| 持续性窗口 | 近20日 | 阳阴比例统计周期 |
| 最小连阳天数 | 3天 | 持续性中连阳条件 |
| 均线周期 | MA20 | 初筛过滤线 |
| 放量阈值 | >1.0 (涨均量>跌均量) | 量能B条件 |
| 试探仓位 | 1/6 ~ 1/3 | 状态3仓位 |
| 标准仓位 | 1.0 | 状态4仓位 |
| 加仓上限 | 1.5 | 状态5企稳加仓上限 |
| 漏斗第一层板块数 | 60~90 | 行业板块指数总数 |
| ETF直筛池 | 100~200 | 类型A+B ETF总数 |

所有参数均基于经验设定, 后续必须通过回测验证和校准。

**采用的路线: 先固化设计 → 参数用经验值 → 实现功能 → 进入回测流程迭代校准。**
选择此路线的原因: 如果先写回测框架再验证参数, 周期太长为时过早; 当前阶段需要的是快速启动、将框架落地。回测环节作为本系统完成后的下一步, 不在当前实现范围内。

## 7. 关键边界场景处理

### 7.1 反弹 vs 翻转

| 维度 | 反弹 (状态2) | 翻转 (状态3) |
|------|------------|------------|
| 结构 | 未突破前高 | 放量突破前高 |
| 量能 | 缩量上涨 | 放量上涨 |
| 持续性 | 短命 (3-5天) | 持续且回调不破前低 |
| 操作 | 不动 | 1/6试探 |

### 7.2 上涨趋势中的正常回调 vs 趋势结束

| 维度 | 正常回调 (状态5) | 趋势结束 (状态3') |
|------|----------------|-----------------|
| 结构 | 未破前低 | 放量跌破前低 |
| 量能 | 缩量下跌 | 放量下跌 |
| 操作 | 等企稳加仓 | 减至1/3防守 |

### 7.3 震荡市

若板块或个股高低点均在一个区间内反复 (无方向性结构), 则所有标的均不满足条件A, 不会进入上涨趋势判定。仓位保持为0或维持不变。

## 8. 策略适用范围

| 维度 | 范围 |
|------|------|
| 市场 | A股 (含 ETF) |
| 标的类型 | 股票、ETF、板块指数、题材指数 |
| 时间周期 | 日线级别 (中线为主, 持仓周期1-6个月) |
| 不适用 | 超短线(日内/1-2日)、宽基ETF、策略因子ETF |
| 市场环境 | 上涨市和震荡市可用, 单边下跌市系统会自然输出空仓 |

### 9.12 数据流: 从引擎到页面

```

    │
    ├─ trend_snapshot_{date}.json     ← 每日全量快照
    │   ├─ market_overview: {}        市场概览
    │   ├─ sectors: []                板块层 (状态+条件+为什么)
    │   ├─ themes: []                 题材层

    │   ├─ etfs: []                   ETF层
    │   ├─ key_actions: []            关键操作点
    │   ├─ scenario_plan: []          明日推演
    │   ├─ watchlist: []              观察列表
    │   └─ comparison: {}             对比数据 (vs昨日/3日前/上周)
    │
    ▼

    ├─ 加载当日 JSON
    ├─ 支持日期切换 → 加载对应日期的 JSON
    └─ 所有"为什么"从 JSON 字段直接渲染, 不需要二次计算
```

### 9.13 技术实现说明

- HTML 单文件, 无后端依赖, 浏览器直接打开
- 每个交易日的 JSON 快照文件存放在 `dashboard/data/` 目录
- 趋势引擎每日运行一次, 产出当日快照 JSON
- HTML 页面通过 fetch 加载本地 JSON 文件渲染
- 日期切换 = 切换加载的 JSON 文件
- 持仓数据手动维护, 存储在 `dashboard/positions.json` 中
- 外部链接均为新标签页打开
